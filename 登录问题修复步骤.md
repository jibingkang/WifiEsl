# 登录问题修复步骤

## 问题现象
1. 后端登录成功（从日志可以看到获取了JWT token）
2. 前端提示登录失败，无法进入管理系统

## 问题分析
前后端数据格式不匹配或前端响应处理逻辑有问题。

### 后端响应流程
1. `proxy_login()` 返回：
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 7200,
  "user": {
    "id": "69cb8e82d33300da21f40834",
    "username": "admin",
    "role": "admin",
    "avatar": "",
    "apiKey": "cef24a6d..."
  }
}
```

2. API包装层 (`auth.py`) 返回：
```json
{
  "code": 20000,
  "message": "登录成功",
  "data": { ...上面的proxy_login返回值... }
}
```

### 前端处理流程
1. axios拦截器 (`index.ts`)：
   - 检查 `res.code && res.code !== 20000` → 显示错误
   - 返回 `res.data ?? res`

2. authStore (`auth.ts`)：
   - 需要处理不同格式的响应

## 已实施的修复

### 1. 修复 authStore.login() 方法
- 添加了对多种响应格式的支持
- 添加了详细的日志输出
- 统一了token和用户信息的提取逻辑

### 2. 修改 axios 拦截器
- 对 `/auth/login` 接口返回原始响应（不进行 `res.data ?? res` 处理）
- 添加了调试日志

### 3. 优化登录页面
- 添加了更多调试日志
- 改进了错误处理

## 如果仍有问题

### 检查点1：后端响应格式
检查浏览器 Network 标签页中 `/auth/login` 请求的响应体，确认格式是：
```json
{
  "code": 20000,
  "message": "登录成功",
  "data": {
    "token": "...",
    "expiresIn": 7200,
    "user": {...}
  }
}
```

### 检查点2：前端错误信息
查看控制台错误信息，特别是：
- 是否有 `登录失败: 未获取到认证令牌` 错误
- 是否有其他未捕获的异常

### 检查点3：CORS 问题
确保后端CORS配置正确，前端可以正常访问API。

## 回滚方案
如果修改后问题更严重，可以：
1. 恢复 `authStore.login()` 到原始简单版本
2. 移除axios拦截器对登录接口的特殊处理
