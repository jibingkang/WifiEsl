# WifiEsl 云服务器部署实施文档

## 一、环境准备

### 1.1 服务器要求

| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| CPU | 2核 | 4核 |
| 内存 | 4GB | 8GB |
| 磁盘 | 50GB SSD | 100GB SSD |
| 带宽 | 5Mbps | 10Mbps |
| 操作系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |

### 1.2 安全组/防火墙配置

| 端口 | 用途 | 说明 |
|------|------|------|
| 22 | SSH | 服务器管理 |
| 80 | HTTP | 前端访问（可转发到443） |
| 443 | HTTPS | 前端+后端API访问 |
| 8000 | 后端API | 开发环境或内部通信 |

## 四、后端部署

### 4.1 创建 Python 虚拟环境

```bash
cd /opt/wifiesl
python3.11 -m venv venv
source venv/bin/activate
```

### 4.2 安装依赖

```bash
cd /opt/wifiesl/backend
pip install --upgrade pip
pip install -r requirements.txt
```

### 4.3 配置环境变量

```bash
cd /opt/wifiesl/backend

# 创建 .env 文件
cat > .env << 'EOF'
# 数据库
DATABASE_URL=sqlite+aiosqlite:///./data/wifi_esl.db

# JWT 密钥（生产环境必须修改！）
SECRET_KEY=your-super-secret-key-change-this-in-production

# 管理员账号
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-secure-password

# WIFI 系统配置（根据实际环境填写）
WIFI_BASE_URL=http://your-wifi-system-ip:8080
WIFI_API_KEY=your-wifi-api-key

# 日志级别
LOG_LEVEL=INFO
EOF
```

### 4.4 初始化数据库

```bash
cd /opt/wifiesl/backend
mkdir -p data

# 运行数据库迁移（如果有）
python -c "import asyncio; from services.db_service import init_db; asyncio.run(init_db())"
```

### 4.5 测试后端启动

```bash
cd /opt/wifiesl/backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

访问 `http://your-server-ip:8000/docs` 验证 API 文档是否正常。

按 `Ctrl+C` 停止测试。

## 六、Nginx 配置

### 6.1 安装 Nginx

```bash
apt install -y nginx
systemctl enable nginx
```

### 6.2 创建 Nginx 配置文件

```bash
cat > /etc/nginx/sites-available/wifiesl << 'EOF'
server {
    listen 80;
    server_name your-domain.com;
    
    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL 证书（使用 Let's Encrypt 或自签名证书）
    ssl_certificate /etc/nginx/ssl/wifiesl.crt;
    ssl_certificate_key /etc/nginx/ssl/wifiesl.key;
    
    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;

    # 前端静态文件
    location / {
        root /opt/wifiesl/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # WebSocket 支持（如果需要）
    location /ws {
        proxy_pass http://127.0.0.1:8000/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 日志
    access_log /var/log/nginx/wifiesl-access.log;
    error_log /var/log/nginx/wifiesl-error.log;
}
EOF
```

### 6.3 启用站点

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/wifiesl /etc/nginx/sites-enabled/

# 删除默认站点
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 创建 SSL 目录
mkdir -p /etc/nginx/ssl
```

### 6.4 配置 SSL 证书

**方式一：使用 Let's Encrypt（推荐）**

```bash
# 安装 Certbot
apt install -y certbot python3-certbot-nginx

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期测试
certbot renew --dry-run
```

**方式二：使用自签名证书（测试环境）**

```bash
# 生成自签名证书
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/nginx/ssl/wifiesl.key \
    -out /etc/nginx/ssl/wifiesl.crt \
    -subj "/C=CN/ST=State/L=City/O=Organization/CN=your-domain.com"
```

### 6.5 启动 Nginx

```bash
systemctl restart nginx
systemctl status nginx
```

## 七、Supervisor 进程管理

### 7.1 安装 Supervisor

```bash
apt install -y supervisor
systemctl enable supervisor
```

### 7.2 创建后端服务配置

```bash
cat > /etc/supervisor/conf.d/wifiesl-backend.conf << 'EOF'
[program:wifiesl-backend]
directory=/opt/wifiesl/backend
command=/opt/wifiesl/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
autostart=true
autorestart=true
startretries=3
user=root
redirect_stderr=true
stdout_logfile=/opt/wifiesl/logs/backend.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=10
environment=PATH="/opt/wifiesl/venv/bin:%(ENV_PATH)s"
EOF
```

### 7.3 创建日志目录

```bash
mkdir -p /opt/wifiesl/logs
```

### 7.4 启动服务

```bash
# 重载 Supervisor 配置
supervisorctl reread
supervisorctl update

# 启动后端服务
supervisorctl start wifiesl-backend

# 查看状态
supervisorctl status
```

### 7.5 常用命令

```bash
# 查看日志
tail -f /opt/wifiesl/logs/backend.log

# 重启后端
supervisorctl restart wifiesl-backend

# 停止后端
supervisorctl stop wifiesl-backend
```

## 八、系统服务配置（可选）

### 8.1 创建 Systemd 服务（替代 Supervisor）

```bash
cat > /etc/systemd/system/wifiesl-backend.service << 'EOF'
[Unit]
Description=WifiEsl Backend Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/wifiesl/backend
Environment=PATH=/opt/wifiesl/venv/bin
ExecStart=/opt/wifiesl/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

# 启用并启动
systemctl daemon-reload
systemctl enable wifiesl-backend
systemctl start wifiesl-backend
systemctl status wifiesl-backend
```

## 十、备份与维护

### 10.1 数据库备份

```bash
# 创建备份脚本
cat > /opt/wifiesl/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/wifiesl/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

# 备份 SQLite 数据库
cp /opt/wifiesl/backend/data/wifi_esl.db $BACKUP_DIR/wifi_esl_$DATE.db

# 保留最近 30 天的备份
find $BACKUP_DIR -name "*.db" -mtime +30 -delete

echo "Backup completed: $BACKUP_DIR/wifi_esl_$DATE.db"
EOF

chmod +x /opt/wifiesl/backup.sh

# 添加到定时任务（每天凌晨 2 点备份）
echo "0 2 * * * /opt/wifiesl/backup.sh >> /var/log/wifiesl-backup.log 2>&1" | crontab -
```

### 10.2 日志清理

```bash
# 创建日志清理脚本
cat > /opt/wifiesl/clean-logs.sh << 'EOF'
#!/bin/bash
# 清理 Nginx 日志
find /var/log/nginx -name "*.log" -mtime +30 -delete

# 清理后端日志
find /opt/wifiesl/logs -name "*.log" -mtime +30 -delete

echo "Logs cleaned at $(date)"
EOF

chmod +x /opt/wifiesl/clean-logs.sh

# 每周清理一次
echo "0 3 * * 0 /opt/wifiesl/clean-logs.sh >> /var/log/wifiesl-cleanup.log 2>&1" | crontab -
```

## 十二、故障排查

### 12.1 常见问题

| 问题 | 排查方法 |
|------|---------|
| 502 Bad Gateway | 检查后端服务是否运行：`supervisorctl status` |
| 前端空白页 | 检查 `dist` 目录是否存在，Nginx 配置是否正确 |
| API 请求失败 | 检查后端日志，`/opt/wifiesl/logs/backend.log` |
| SSL 证书错误 | 检查证书路径和有效期：`openssl x509 -in /etc/nginx/ssl/wifiesl.crt -noout -dates` |
| 权限错误 | 检查文件权限：`chown -R www-data:www-data /opt/wifiesl/frontend/dist` |

### 12.2 查看日志

```bash
# Nginx 错误日志
tail -f /var/log/nginx/wifiesl-error.log

# 后端日志
tail -f /opt/wifiesl/logs/backend.log

# Supervisor 日志
tail -f /var/log/supervisor/supervisord.log
```

### 12.3 重启所有服务

```bash
# 一键重启脚本
cat > /opt/wifiesl/restart.sh << 'EOF'
#!/bin/bash
echo "Restarting WifiEsl services..."
supervisorctl restart wifiesl-backend
systemctl restart nginx
echo "Services restarted at $(date)"
EOF

chmod +x /opt/wifiesl/restart.sh
/opt/wifiesl/restart.sh
```

## 十三、安全加固（可选）

### 13.1 禁用 root 登录

```bash
# 创建普通用户
useradd -m -s /bin/bash wifiesl
usermod -aG sudo wifiesl

# 设置密码
passwd wifiesl

# 禁用 root 登录
sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
systemctl restart sshd
```

### 13.2 配置防火墙

```bash
# 安装 UFW
apt install -y ufw

# 默认拒绝
ufw default deny incoming
ufw default allow outgoing

# 允许 SSH、HTTP、HTTPS
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp

# 启用防火墙
ufw enable

# 查看状态
ufw status
```

```bash
apt install -y fail2ban
systemctl enable fail2ban

```
