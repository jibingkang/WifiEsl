# 模板管理功能说明

## 功能概述
实现了完整的模板管理功能，支持：
- 查看所有模板列表
- 新增、编辑、删除模板
- JSON文件导入模板
- 手动输入JSON数据创建模板

## 访问方式
1. **侧边栏菜单**：新增了"模板管理"菜单项，位于"模板更新"下方
2. **模板更新页面**：顶部添加了"前往模板管理"的链接
3. **直接访问**：访问 `/template/manage` 路由

## 功能详解

### 1. 模板列表
- 以卡片形式展示所有模板
- 显示模板名称、描述、屏幕类型、字段数量
- 字段预览显示每个字段的key、label、类型、必填状态、默认值
- 支持编辑和删除操作

### 2. 新增模板
- **手动创建**：填写模板ID、名称、描述、屏幕类型
- **字段管理**：动态添加/删除字段，配置字段属性（key、label、类型、必填、默认值、占位符等）

### 3. JSON导入
- **文件上传**：支持上传`.json`格式的模板文件
- **手动输入**：支持直接粘贴JSON数据

### 4. JSON格式支持
支持两种JSON格式：

#### 格式A：简化格式（从template.json示例）
```json
{
  "tid": "69d7c56edc4d8169e2140f99",
  "tname": "Test1",
  "data": {
    "text-0": "TEXT"
  }
}
```
系统会自动将`data`对象转换为字段定义。

#### 格式B：完整格式
```json
{
  "tid": "template-id",
  "tname": "模板名称",
  "description": "模板描述",
  "screen_type": "monochrome",
  "fields": [
    {
      "key": "text-0",
      "label": "文本字段",
      "type": "text",
      "required": true,
      "default_value": "默认文本"
    }
  ]
}
```

### 5. 后端API
调用的后端API端点：
- `GET /api/v1/templates` - 获取模板列表
- `POST /api/v1/templates` - 创建新模板
- `PUT /api/v1/templates/{tid}` - 更新模板
- `DELETE /api/v1/templates/{tid}` - 删除模板

## 使用示例

### 1. 通过template.json示例创建模板
1. 在项目根目录找到 `template.json` 文件
2. 在模板管理页面点击"导入JSON文件"
3. 选择 `template.json` 文件
4. 系统会自动解析并创建模板

### 2. 手动创建复杂模板
1. 点击"新增模板"
2. 填写：
   - 模板ID: `product-price-tag`
   - 名称: `商品价格标签`
   - 描述: `用于显示商品名称和价格的模板`
   - 屏幕类型: `color`（彩色屏）
3. 添加字段：
   - 字段1: key=`product-name`, label=`商品名称`, type=`text`, 必填
   - 字段2: key=`price`, label=`价格`, type=`text`, 必填, 默认值=`¥0.00`
   - 字段3: key=`barcode`, label=`条形码`, type=`image`, 非必填
4. 点击"创建"

### 3. 编辑现有模板
1. 在模板列表中点击要修改模板的"编辑"按钮
2. 修改模板信息或字段
3. 点击"更新"

## 注意事项

1. **模板ID唯一性**：创建时需确保tid唯一
2. **字段Key唯一性**：同一模板内字段key不能重复
3. **必填字段**：在模板更新向导中，必填字段必须有值
4. **JSON格式验证**：上传的JSON需符合格式要求
5. **删除确认**：删除模板前有二次确认提示

## 问题排查

如果遇到问题：

1. **模板列表为空**：检查后端API是否正常工作
2. **JSON导入失败**：检查JSON格式是否正确
3. **字段显示异常**：刷新页面或检查网络连接
4. **操作无响应**：检查浏览器控制台错误信息
