# 登录问题修复说明

## 问题描述
用户报告登录时报错：`userStore.login is not a function`

## 问题分析
1. **根本原因**：登录页面（`LoginView.vue`）错误地调用了 `userStore.login()` 方法
2. **代码位置**：`frontend/src/views/login/LoginView.vue` 第127行
3. **问题细节**：
   - `userStore` 是用户管理相关的 store，不包含登录功能
   - 登录功能应该在专门的 `authStore` 或直接调用 `authApi`

## 修复内容

### 1. 创建认证状态管理 Store
新建 `frontend/src/stores/auth.ts`，包含：
- `login(username, password)` - 用户登录
- `logout()` - 用户登出
- `fetchUserInfo()` - 获取用户信息
- `checkAuth()` - 检查登录状态
- `getUserRole()` - 获取用户角色
- `hasPermission(requiredRole)` - 检查权限

### 2. 修复登录页面
修改 `frontend/src/views/login/LoginView.vue`：
- 移除对 `userStore` 的导入
- 添加对 `authStore` 的导入
- 修改 `handleLogin()` 函数调用 `authStore.login()`

### 3. 修复路由守卫
修改 `frontend/src/router/routes.ts`：
- 将路由守卫中使用的 `userStore` 改为 `authStore`
- 更新认证检查逻辑，使用 `isAuthenticated` 状态

### 4. 统一 localStorage 键名
- 将 token 存储键名统一为 `'wifi_esl_token'`（与 axios 拦截器保持一致）
- 将用户信息存储键名统一为 `'wifi_esl_user'`

### 5. 更新 Store 导出
修改 `frontend/src/stores/index.ts`：
- 添加 `export { useAuthStore } from './auth'`

## 技术细节

### 登录流程
1. 用户输入用户名密码
2. 调用 `authStore.login(username, password)`
3. `authStore` 调用 `authApi.login(params)`
4. `authApi` 发送 POST 请求到 `/api/v1/auth/login`
5. 后端处理登录，返回 token 和用户信息
6. `authStore` 保存 token 到 localStorage
7. 设置 `isAuthenticated = true`
8. 跳转到首页或目标页面

### Token 存储
- **localStorage 键名**：`wifi_esl_token`
- **用户信息键名**：`wifi_esl_user`
- **自动注入**：axios 拦截器自动将 token 添加到请求头

### 路由保护
- 所有需要认证的路由默认开启 `requiresAuth: true`
- 登录页面设置 `requiresAuth: false` 避免重定向循环
- 未登录用户访问受保护路由时，自动跳转到登录页
